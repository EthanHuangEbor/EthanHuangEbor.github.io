# LLM-Wiki 与个人知识图谱

我目前能稳定检索到的公开资料，是后续 LLM-Wiki 论文对 Karpathy “LLM Wiki”概念的引用和展开。把它放到个人主页里，可以理解为一句话：不要只把材料上传，而要把材料编译成 LLM 能读、能引用、能沿链接继续追踪的 Wiki。

## 核心概念

LLM-Wiki 不是普通资料夹，也不是只靠向量检索的黑箱笔记库。它更像一套知识页协议：

- 每个节点回答一个具体问题。
- 每个节点保留来源、结论、证据和边界。
- 节点之间用显式双链连接，而不是只靠全文搜索。
- 错题、实验失败和概念误解进入 Error Book，成为下一轮学习的训练样本。

## 对我的主页意味着什么

知识图谱页以后不只是“展示我写过什么”，而要服务三个动作：

1. 导师能快速看到我在课程、项目和论文中沉淀出的可复用理解。
2. 我自己能把手写推导、刷题网页、项目调试和论文阅读统一整理。
3. Codex 或其他 LLM 能根据这些 Wiki 页做检索、归因、评分和复习计划。

## Wiki 页模板

后续每篇正式笔记尽量包含：

| 字段 | 用途 |
| --- | --- |
| Problem | 这篇笔记解决什么问题 |
| Context | 来自哪门课、哪篇论文、哪个项目或哪次刷题 |
| Claim | 当前结论是什么 |
| Evidence | 公式、题目、实验截图、代码结果或参考文献 |
| Boundary | 什么情况下这个结论不成立 |
| Links | 至少 2 个指向已有节点的双链 |
| Error Book | 曾经错在哪里、以后如何避免 |

## 手写笔记进入 LLM-Wiki 的路径

清晰手写页优先 OCR，再人工校对成 Markdown。复杂推导、草图和临时演算不强行全文转写，只保留图片、3-5 行摘要、关键词和关联节点。这样做的重点不是“把所有字都识别出来”，而是让未来检索时能知道这张图在回答什么问题。

## 错题和刷题网页

类似 [[概率统计前9章刷题记录]] 这种网页，最有价值的不是题库本身，而是完成作答后导出的错误分布。后续每次刷题可以沉淀三类节点：

- 概念误解：例如把“不能拒绝原假设”写成“接受原假设”。
- 公式误用：例如中心极限定理中总和与均值的标准化分母混淆。
- 题型风险：例如填空题暴露记忆问题，解答题暴露推导步骤问题。

## 与已有流程的关系

这篇笔记是 [[学习笔记沉淀流程]] 的升级版：原流程解决“怎么收集和发布”，LLM-Wiki 解决“每个节点如何对 LLM 友好”。如果每篇笔记都能被搜索、被引用、被链接、被纠错，知识图谱就会逐渐从展示页变成长期学习系统。
